# AI 專案秘書平台｜產品功能介紹

> **文件版號**：V1.0  
> **建立日期**：2026-01-10

---

## 產品概述

**AI 專案秘書**是一個以對話驅動的專案管理助理平台，專為專案經理 (PM) 和團隊成員設計。透過 AI 技術，將會議記錄、LINE 訊息、Email、文件等碎片化資訊，自動轉化為可追蹤的任務與行動項目。

### 核心價值
- 🤖 **AI 驅動**：智慧識別任務、自動分類、建議負責人與期限
- 📋 **統一管理**：一站式管理專案工作、功能模組、待辦事項
- 🔗 **證據溯源**：所有任務都可追溯到原始來源（Citation）
- ⏰ **主動提醒**：逾期預警、卡關追蹤、待回覆提示

---

## 系統功能架構

### 主要入口（左側導覽）

| 入口 | 功能說明 |
|------|----------|
| **儀表板** | 晨間簡報、需要處理的事項、快速概覽 |
| **收件匣** | AI 建議暫存區、人工確認入庫 |
| **任務清單** | 多視圖任務管理中心 |
| **來源文件** | 上傳文件與證據管理 |
| **設定** | 專案設定與系統設定 |

---

## 一、儀表板 (Dashboard)

儀表板是每日工作的起點，提供專案的即時概況。

### 功能特色

#### 1. 晨間簡報
- AI 自動生成每日專案摘要
- 重點提示今日待處理事項
- 呈現專案整體進度

#### 2. 需要處理
- **逾期任務**：已超過期限的任務清單
- **即將到期**：未來 3 天內到期的任務
- **卡關事項**：被標記為卡關的任務
- **待回覆**：等待外部回覆的追蹤項

#### 3. AI 秘書對話
- 自然語言查詢專案資訊
- 快速建立任務與待辦
- 智慧推薦下一步行動

---

## 二、收件匣 (Inbox)

收件匣是 AI 建議進入系統的「緩衝區」，確保人工確認後才正式入庫。

### 運作流程

```
輸入來源 → AI 分析 → 建議卡片 → 人工確認 → 入庫
```

### 功能特色

#### 1. 多元輸入
- 貼上文字（會議記錄、LINE/Email 內容）
- 上傳檔案（文件、圖片、Excel）
- 輸入連結

#### 2. AI 智慧分析
- 自動識別任務類型
- 建議負責人與期限
- 提取關鍵資訊

#### 3. 建議卡片操作
- **確認**：直接入庫為正式任務
- **編輯後確認**：修改後再入庫
- **拒絕**：不入庫（保留來源記錄）

---

## 三、任務清單 (Tasks)

任務清單是系統的核心功能區，採用多視圖設計，用不同角度管理同一批任務資料。

### 三大任務類別

| 類別 | 說明 | 專屬欄位 |
|------|------|----------|
| **專案工作** | 專案層級的工作項目 | 無 |
| **功能模組** | 系統功能開發項目 | 需求規格、功能規格(FRD)、設計雛形 |
| **待辦事項** | 一般待辦與行動項目 | 無 |

### 視圖 TAB

| TAB | 說明 |
|-----|------|
| 專案工作 | WBS 結構化專案工作視圖 |
| 功能模組 | 功能模組開發管理 |
| 待辦事項 | 一般待辦清單 |
| 待確認 | 需要確認的事項 |
| 變更請求 | 需求變更追蹤 |
| 決議 | 已確認的決議事項 |

### 功能特色

#### 1. 類別轉換
- 三種類別可互相轉換
- 功能模組有專屬欄位保護（有內容時禁止轉換）
- 轉換後自動導向對應視圖

#### 2. 層級結構（WBS）
- 支援父子任務關係
- 樹狀結構展開/收合
- 自動 WBS 編號提取

#### 3. 拖曳排序
- 拖曳調整任務順序
- 拖曳變更父子關係
- 即時視覺回饋

#### 4. 任務狀態管理

| 狀態 | 說明 |
|------|------|
| 未開始 | 任務尚未啟動 |
| 進行中 | 任務執行中 |
| 卡關 | 遇到障礙無法繼續 |
| 待回覆 | 等待外部回覆 |
| 暫停 | 任務暫時擱置 |
| 已完成 | 任務已完成 |

---

## 四、功能模組詳情

功能模組提供專屬的詳情頁面，用於管理系統功能開發的完整生命週期。

### 專屬欄位

#### 1. 需求規格
- 使用 Markdown 格式記錄
- 支援列表、標題等格式
- 可累積多次討論內容

#### 2. 功能規格 (FRD)
- 詳細功能規格文件
- 支援雲端文件連結（Google Docs 等）
- Markdown 格式編輯

#### 3. 設計雛形
- 多個雛形項目管理
- 支援圖片上傳/拖放
- 連結 Figma 等設計工具
- 每個雛形可附說明文字

---

## 五、來源文件 (Sources)

來源文件用於管理所有輸入的原始資料，是證據鏈（Citation）的基礎。

### 功能特色

#### 1. 文件類型支援
- 文字內容（貼上）
- 文件檔案（PDF、Word、Excel 等）
- 圖片與截圖
- 外部連結

#### 2. 內容保護
- 來源內容不可修改（維持證據真實性）
- 可修改顯示名稱與描述

#### 3. 關聯查詢
- 查看引用此來源的所有任務
- 一鍵跳轉到關聯項目

## 六、設定 (Settings)

### 專案設定

專案管理者可配置：
- 專案基本資訊（名稱、描述）
- 成員管理（邀請、角色指派）
- 通知規則

### 系統設定

系統管理員可配置：
- AI 供應商與模型選擇
- API Key 設定（加密儲存）
- Supabase 連線配置

## 技術特點

### 部署架構
- **前端**：React + TypeScript + Vite
- **後端**：Supabase (PostgreSQL + Edge Functions)
- **AI**：OpenAI GPT-4o-mini / text-embedding-3-small

### 配置優先順序
1. **環境變數**（Vercel 部署）
2. **localStorage**（本地開發）

部署到 Vercel 後，團隊成員登入即可使用，無需手動配置。

## 使用情境

### 情境 1：會議記錄整理
1. 會議結束後，貼上逐字稿到收件匣
2. AI 自動識別並建立多張建議卡片
3. 確認後，任務自動分配到對應的專案工作

### 情境 2：需求變更追蹤
1. 收到客戶變更需求，輸入到系統
2. AI 識別為變更請求（CR）
3. 在「變更請求」TAB 追蹤處理進度

### 情境 3：功能開發管理
1. 建立功能模組任務
2. 填寫需求規格與功能規格
3. 上傳設計雛形
4. 追蹤開發進度直到完成
