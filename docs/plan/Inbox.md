# 模組：收件匣（Inbox）

## 目標
所有 AI 生成結果先進收件匣，人工確認後才入庫，避免誤判與認知落差。

## 主要情境
- 貼上會議逐字稿 → 產生待辦/決議/待回覆建議卡
- 貼上 LINE/Email 訊息 → 產生待回覆或變更建議卡
- 上傳 WBS/RFP → 產生專案工作骨架建議卡

## UI 需求（MVP）
- 建議卡列表（可篩選類型：任務/待回覆/決議/變更/專案工作）
- 每張卡必顯示：類型、摘要、信心度（可選）、Citation
- 操作：Confirm / Edit / Reject
- Confirm 後：轉為正式項目並從 Inbox 移除

## 驗收標準
- 送出輸入後 Inbox 出現建議卡
- Confirm 後卡片消失，且在任務清單相對應視圖可看到新項目
- Edit 後 Confirm：入庫內容以編輯版為準

## 測試案例
- 貼上含「麻煩開權限」訊息 → AI 產生待回覆（對象：外部/客戶）建議卡
- 貼上「客戶要改成 SSO」→ AI 產生 CR 高風險建議卡
