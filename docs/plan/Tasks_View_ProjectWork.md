# 任務清單視圖：專案工作（Project Work）

## 目標
管理交付骨架：功能模組/頁面、交付文件、會議、外部依賴等。

## 主要情境
- PM 上傳 WBS v1 → AI 提取成專案工作建議卡 → Inbox Confirm 入庫
- PM 在專案工作上指派 Owner 或拆解任務
- 團隊成員也可新增/指派專案工作（如：需要 API、需要授權、需要驗收會議）

## UI 需求（MVP）
- 清單顯示：工作名稱、類型（功能/文件/會議/外部依賴）、Owner、到期、進度摘要
- 進度摘要為彙總：此專案工作底下的任務完成比例/是否 blocked/overdue
- 支援：指派 Owner、設定期限、建立/連結任務、查看 Citation

## 驗收標準
- 從 Inbox 入庫的專案工作可在此視圖看到
- 指派後，受指派者在「我的任務」能看到對應任務
