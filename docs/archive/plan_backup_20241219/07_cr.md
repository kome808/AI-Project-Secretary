# Module 07：Change Request (CR) (需求變更控管)

## 1. 模組目標
記錄並追蹤超出原始規格基準的變更項目。透過評估風險、影響範圍與決策過程，確保變更透明化且具備完整證據鏈。

## 2. 功能需求 (Requirements)

### 2.1 CR 列表與詳情
- **基本欄位**：標題、摘要、提案人、風險等級 (高/中/低)。
- **狀態流轉**：已請求 (Requested) → 審核中 (Reviewing) → 已核准 (Approved) → 已實作 (Implemented) / 已拒絕 (Rejected)。
- **影響分析**：關聯受影響的模組 (Modules) 與頁面 (Pages)。

### 2.2 變更追蹤
- **證據 Citation**：連結至客戶要求變更的原始對話或文件。
- **關聯待辦**：一鍵產生因變更而產生的後續 Action Items。

## 3. 技術實作規劃

### 3.1 資料結構
```typescript
interface CRMeta {
  cr_status: string;
  risk_level: 'low' | 'medium' | 'high';
  impact_modules: string[];
  impact_pages: string[];
}
```

## 4. 驗收標準
1. 高風險 CR 在儀表板有特別標示。
2. CR 必須具備原始來源證據。
3. 支援依據狀態與風險等級篩選。
